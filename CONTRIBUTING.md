# Contributing Guide

Thank you for helping improve the Deepfake Detection System! This project is educational and privacy-preserving. Please review the safety rules before contributing.

## Development Workflow
1. Fork and clone the repository.
2. Create a virtual environment and install dependencies with `pip install -r requirements.txt`.
3. Run `pytest` before submitting changes.
4. Follow PEP8 style and add type hints and docstrings.
5. Update documentation (README, docs/architecture.md) when changing functionality.
6. Ensure no real personal data is added to the repo; use synthetic fixtures only.

## Commit Standards
- Use clear, imperative commit messages.
- Keep changes focused and well-tested.
- Include logging and input validation where appropriate.

## Reporting Issues
- Use the issue template (if available) and include reproduction steps.
- Do not upload sensitive media. Provide synthetic examples instead.

## Security
- Review SECURITY.md for disclosure guidelines.
